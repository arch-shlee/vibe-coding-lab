# Checkpoints - 진행 상황 점검

> 프로젝트를 진행하며 각 단계별로 진행 상황을 점검하세요.

## 📋 체크포인트 사용법

### 왜 체크포인트가 필요한가요?

- ✅ 진행 상황을 명확히 파악
- ✅ 빠뜨린 부분이 없는지 확인
- ✅ 학습 목표 달성 여부 점검
- ✅ 다음 단계로 넘어갈 준비 확인

### 사용 방법

1. 각 Phase를 완료할 때마다 체크
2. 모든 항목을 체크하지 않았다면 해당 Phase 다시 검토
3. 막히는 부분은 GUIDE.md 참고
4. 그래도 안 되면 solution/ 참고 (최후의 수단)

## Checkpoint 1: 환경 설정 ✅

### 필수 항목
- [ ] Python 3.9 이상 설치 확인
- [ ] 가상 환경 생성 완료
- [ ] requirements.txt 의존성 설치 완료
- [ ] pytest가 정상 작동하는지 확인

### 확인 방법
```bash
# Python 버전 확인
python --version  # 3.9 이상

# pytest 버전 확인
pytest --version  # 설치되어 있어야 함

# 첫 테스트 실행
pytest test_todo_list.py::test_todo_list_초기화 -v
# PASSED가 나와야 함
```

### 통과 기준
- 모든 명령어가 에러 없이 실행됨
- 첫 테스트가 통과함

## 🔄 개선 계획

### 부족한 부분 확인

각 체크포인트에서 체크하지 못한 항목이 있다면:

1. **왜 체크하지 못했나요?**
   - 어려워서?
   - 시간이 부족해서?
   - 이해가 안 되어서?

2. **어떻게 개선할까요?**
   - 다시 시도하기
   - GUIDE.md 재확인
   - AI에게 질문
   - 커뮤니티에 도움 요청

3. **학습 계획**
   - 부족한 부분 재학습
   - 비슷한 프로젝트 한 번 더
   - Level 2로 진행하기 전에 완벽히 이해
