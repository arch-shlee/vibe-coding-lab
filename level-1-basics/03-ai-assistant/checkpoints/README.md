# 체크포인트

> 학습 진행 상황을 확인하세요.

## 📋 사용 방법

1. 각 체크포인트를 순서대로 진행
2. 모든 항목을 완료하면 ✅ 표시
3. 막히는 부분은 다시 복습
4. 모든 체크포인트 통과 시 다음 모듈로

---

## Checkpoint 1: 프롬프트 엔지니어링 기초

### 이론 이해
- [ ] 좋은 프롬프트의 3요소를 설명할 수 있다
  - [ ] 명확한 목표 (What)
  - [ ] 충분한 컨텍스트 (Why/How)
  - [ ] 구체적인 제약사항 (Constraints)
- [ ] 나쁜 프롬프트와 좋은 프롬프트를 구분할 수 있다

### 실습
- [ ] `prompt_practice.md`의 5개 연습 완료
- [ ] 각 프롬프트를 실제로 AI에게 보내봄
- [ ] 개선 전후의 차이를 확인함

### 자가 평가
다음 상황에 대한 좋은 프롬프트를 작성할 수 있나요?
- [ ] 새 기능 구현 요청
- [ ] 버그 수정 요청
- [ ] 코드 리팩토링 요청

## Checkpoint 2: AI 활용 전략

### AI가 강한 상황 이해
- [ ] 반복적인 코드 작성에 활용할 수 있다
- [ ] 보일러플레이트 코드 생성에 활용할 수 있다
- [ ] 테스트 코드 작성에 활용할 수 있다
- [ ] 코드 설명 및 문서화에 활용할 수 있다

### AI 사용 시 주의사항
- [ ] 보안 관련 코드는 검증이 필요함을 안다
- [ ] AI 코드를 이해하지 못하면 사용하지 않는다
- [ ] 아키텍처 결정은 직접 한다

### 실전 판단
다음 상황에서 AI를 어떻게 활용할지 판단할 수 있나요?
- [ ] CRUD API 엔드포인트 여러 개 만들기
- [ ] 복잡한 비즈니스 로직 구현
- [ ] 보안이 중요한 인증 시스템
- [ ] 프로젝트 아키텍처 결정

---

## Checkpoint 3: 효과적인 협업

### 대화 스킬
- [ ] 명확한 초기 요청을 할 수 있다
- [ ] AI의 응답을 읽고 이해한다
- [ ] 구체적인 피드백을 줄 수 있다
- [ ] "왜?"를 물어볼 수 있다
- [ ] 대안을 요청할 수 있다

### 반복적 개선
- [ ] 한 번에 완벽을 기대하지 않는다
- [ ] 대화를 통해 점진적으로 개선한다
- [ ] 만족할 때까지 피드백을 반복한다

### 실습 확인
다음을 경험했나요?
- [ ] AI에게 요청 → 응답 → 피드백 → 개선 (2회 이상)
- [ ] "왜?"를 물어보고 설명을 들음
- [ ] 다른 방법을 물어보고 비교함
