# 03 - AI Assistant: AI와 함께 코딩하기

> AI를 효과적으로 활용하여 더 나은 코드를 더 빠르게 작성하는 방법을 배웁니다.

## 🎯 학습 목표

- 프롬프트 엔지니어링 기초 익히기
- AI와 효과적으로 협업하는 전략 배우기
- 좋은 질문과 나쁜 질문 구분하기
- AI의 강점과 한계 이해하기
- 반복적인 개선(iterative refinement) 방법 실습하기

**예상 소요 시간**: 3-4시간

## 📚 AI 코딩 어시스턴트란?

### AI의 역할

AI 코딩 어시스턴트(Claude, ChatGPT, Copilot 등)는:

**할 수 있는 것**:
- 코드 작성 및 리팩토링
- 버그 찾기 및 수정
- 코드 설명 및 문서화
- 테스트 코드 작성
- 알고리즘 제안
- 코드 리뷰

**할 수 없는 것**:
- 프로젝트의 방향 결정
- 비즈니스 요구사항 이해
- 완벽한 코드 보장
- 당신을 대신해서 생각하기

### 핵심 원칙

**1. AI는 도구, 당신이 개발자**
```
당신: 무엇을 만들지 결정
AI: 어떻게 만들지 제안
당신: 최종 결정 및 검증
```

**2. 명확한 의도 전달이 핵심**
- 모호한 요청 = 모호한 결과
- 구체적인 요청 = 정확한 결과

**3. 반복적인 개선**
- 한 번에 완벽할 수 없음
- 대화를 통해 점진적으로 개선

## 💬 프롬프트 엔지니어링 기초

### 좋은 프롬프트의 3요소

#### 1. 명확한 목표 (What)

❌ **나쁜 예시**:
```
함수 만들어줘
```

✅ **좋은 예시**:
```
사용자 나이를 입력받아서 성인인지 판별하는 함수를 만들어줘.
함수명은 is_adult로 하고, 19세 이상이면 True를 반환해야 해.
```

#### 2. 충분한 컨텍스트 (Why/How)

❌ **나쁜 예시**:
```
이 코드 고쳐줘
```

✅ **좋은 예시**:
```
이 함수는 리스트의 평균을 계산하는 건데,
빈 리스트를 넣으면 에러가 나.
빈 리스트일 때는 0을 반환하도록 수정해줘.
```

#### 3. 구체적인 제약사항 (Constraints)

❌ **나쁜 예시**:
```
데이터베이스 연결 코드 작성해줘
```

✅ **좋은 예시**:
```
SQLite 데이터베이스에 연결하는 코드를 작성해줘.
- Python 3.10 사용
- context manager 패턴 사용
- 에러 처리 포함
- 주석으로 각 단계 설명
```

## 🎯 AI 활용 전략

### 언제 AI를 활용하면 좋을까?

#### 🟢 AI가 강한 상황

**1. 반복적인 코드 작성**
```
"유사한 CRUD 함수를 4개 더 만들어줘"
```

**2. 보일러플레이트 코드**
```
"FastAPI로 REST API 기본 구조를 만들어줘"
```

**3. 테스트 코드 작성**
```
"이 함수에 대한 pytest 테스트를 작성해줘.
edge case도 포함해서."
```

**4. 코드 설명 및 문서화**
```
"이 함수가 뭘 하는지 docstring으로 설명해줘"
```

**5. 알고리즘 구현**
```
"퀵소트를 Python으로 구현해줘.
주석으로 각 단계 설명도 포함해서."
```

#### 🟡 신중하게 사용해야 할 상황

**1. 복잡한 비즈니스 로직**
- AI가 요구사항을 완벽히 이해하기 어려움
- 단계별로 나누어서 요청

**2. 보안 관련 코드**
- AI가 생성한 코드를 반드시 검증
- 보안 취약점 체크

**3. 성능이 중요한 코드**
- AI의 제안이 최적이 아닐 수 있음
- 프로파일링 필요

#### 🔴 AI를 사용하지 말아야 할 상황

**1. 아키텍처 결정**
- 프로젝트 구조, 기술 스택 선택은 당신의 몫

**2. 비즈니스 의사결정**
- 어떤 기능을 만들지는 AI가 결정할 수 없음

**3. 코드 이해 없이 복사-붙여넣기**
- 이해하지 못한 코드는 사용하지 마세요

## 🤝 효과적인 협업 방법

### 대화의 흐름

#### 1단계: 명확한 요청
```
나: "사용자 인증 시스템을 만들고 싶어.
     JWT 토큰을 사용하고, 로그인/로그아웃 기능이 필요해."
```

#### 2단계: AI 응답 이해
```
AI: [JWT 인증 코드 제공]
```
- 코드를 읽고 이해하기
- 모르는 부분은 질문

#### 3단계: 피드백 및 개선
```
나: "좋은데, 토큰 만료 시간을 24시간으로 설정해줄래?
     그리고 주석을 좀 더 자세히 달아줘."
```

#### 4단계: 검증
```
나: "이 코드에 보안 취약점은 없을까?
     특히 SQL 인젝션이나 XSS 공격에 대해 확인해줘."
```

#### 5단계: 반복
- 만족할 때까지 2-4단계 반복

### 효과적인 질문 패턴

#### "왜?"를 물어보기
```
나: "왜 이 방법을 선택했어?"
AI: [설명]
나: "다른 방법도 있어?"
AI: [대안 제시]
```

#### "만약에?"로 확장하기
```
나: "만약 데이터베이스 연결이 실패하면 어떻게 돼?"
AI: [에러 처리 설명]
나: "그럼 재시도 로직을 추가해줄래?"
```

#### "비교"로 이해 깊이 높이기
```
나: "이 방법과 다른 방법의 차이점은 뭐야?"
AI: [비교 설명]
```

### 피드백 주는 방법

#### ✅ 좋은 피드백
```
"거의 완벽한데, 변수명을 좀 더 명확하게 바꿔줄래?
 예를 들어, 'data'보다는 'user_data'가 좋을 것 같아."
```

#### ❌ 나쁜 피드백
```
"이상해. 다시 해줘."
```
- 무엇이 문제인지 구체적으로 설명

## 💡 실습: AI와 함께 프로젝트 만들기

### 프로젝트: 간단한 TODO 앱

#### 실습 1: 요구사항 정리 및 첫 요청

**목표**: AI에게 TODO 앱의 기본 구조 요청하기

**당신의 역할**:
1. 어떤 기능이 필요한지 정리
2. 기술 스택 결정
3. 프로젝트 구조 결정

**AI의 역할**:
1. 코드 작성
2. 파일 구조 제안
3. 구현 방법 제안

**실습**:
```
안티그라비티에 다음과 같이 요청하세요:

"Python으로 간단한 TODO 앱을 만들고 싶어.
다음 기능이 필요해:
1. TODO 추가
2. TODO 목록 보기
3. TODO 완료 처리
4. TODO 삭제

파일 하나로 만들고, 함수별로 나눠줘.
데이터는 리스트로 메모리에만 저장하면 돼."
```

#### 실습 2: 코드 리뷰 및 개선

**AI가 제공한 코드를 검토하세요**:
- 함수명이 명확한가?
- 주석이 충분한가?
- 에러 처리가 되어 있는가?
- 코드를 이해할 수 있는가?

**개선 요청 예시**:
```
"add_todo 함수에 빈 문자열을 입력하면 어떻게 돼?
 빈 TODO는 추가되지 않도록 검증을 추가해줄래?"
```

#### 실습 3: 테스트 추가

```
"이 TODO 앱의 모든 함수에 대한 pytest 테스트를 작성해줘.
각 함수마다 최소 3개의 테스트 케이스를 포함해줘:
- 정상 케이스
- Edge case
- 에러 케이스"
```

#### 실습 4: 기능 확장

```
"TODO에 우선순위(high, medium, low)를 추가하고 싶어.
기존 코드를 수정해서:
1. TODO 추가 시 우선순위 지정
2. 우선순위별로 정렬해서 보기
3. 기존 테스트는 통과하도록 유지

어떻게 수정하면 좋을까?"
```

## 🎓 학습 정리

### AI와 협업하는 핵심 원칙

**원칙 1: 명확하게 요청하라**
```
모호한 요청 = 모호한 결과
명확한 요청 = 정확한 결과
```

**원칙 2: 이해하고 검증하라**
```
복사-붙여넣기만 하지 마세요.
코드를 읽고, 이해하고, 테스트하세요.
```

**원칙 3: 반복적으로 개선하라**
```
한 번에 완벽할 수 없습니다.
대화를 통해 점진적으로 개선하세요.
```

**원칙 4: AI의 한계를 이해하라**
```
AI는 도구입니다.
최종 결정과 책임은 당신에게 있습니다.
```

### 좋은 프롬프트 체크리스트

작성한 프롬프트가 다음을 포함하는지 확인하세요:

- [ ] 명확한 목표 (무엇을 만들고 싶은지)
- [ ] 충분한 컨텍스트 (왜 필요한지, 어떤 상황인지)
- [ ] 구체적인 요구사항 (어떻게 작동해야 하는지)
- [ ] 제약사항 (기술 스택, 코드 스타일 등)
- [ ] 예시 또는 예상 결과 (있다면)

## 🔍 실전 팁

### 팁 1: 단계별로 접근하기

**한 번에 다 하지 마세요**:
```
❌ "TODO 앱을 만들어줘. 모든 기능 포함."

✅ 단계적 접근:
1. "먼저 TODO 추가 기능만 만들어줘"
2. "이제 목록 보기 기능을 추가해줘"
3. "완료 처리 기능도 추가해줘"
4. ...
```

**예시가 있으면 더 정확합니다**:
```
"사용자 입력을 받아서 이메일 형식인지 검증하는 함수를 만들어줘.

예시:

```

```

### 팁 4: 대안 요청하기

```
"이 문제를 해결하는 다른 방법도 있어?

```

### 팁 5: 설명 요청하기

```
"이 코드가 어떻게 작동하는지 단계별로 설명해줘.

## 🆘 막혔나요?

### 자주 하는 질문

**Q: AI가 내 의도를 잘못 이해했어요**
A: 다시 설명하세요. 예시를 들어주면 더 좋습니다.
```
"아니야, 내가 원한 건 이게 아니야.
 예를 들어, 입력이 [예시]이면 출력이 [예시]가 되어야 해."
```

**Q: AI가 너무 복잡한 코드를 만들었어요**
A: 더 간단하게 요청하세요.
```
"너무 복잡해. 더 간단하게 만들어줄래?
 외부 라이브러리 없이, 기본 문법만 사용해서."
```

**Q: AI의 코드가 작동하지 않아요**
A: 에러 메시지를 공유하세요.
```
"이 코드를 실행하니까 다음 에러가 나:
[에러 메시지 복사]
어떻게 고치면 될까?"
```

**Q: AI가 제안한 방법이 최선인지 모르겠어요**
A: 대안을 물어보세요.
```
"이 방법 말고 다른 접근법도 있어?
각각의 장단점을 비교해줄래?"
```

**Q: 코드는 이해했는데, 왜 이렇게 하는지 모르겠어요**
A: 설명을 요청하세요.
```
"왜 이 방법을 선택했어?
다른 방법과 비교해서 어떤 장점이 있어?"
```
