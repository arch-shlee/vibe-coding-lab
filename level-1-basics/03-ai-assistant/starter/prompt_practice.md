개선된 프롬프트가 다음을 포함하는지 확인하세요:

### 좋은 프롬프트의 3요소
- [ ] **명확한 목표**: 무엇을 만들고 싶은지
- [ ] **충분한 컨텍스트**: 왜 필요한지, 어떤 상황인지
- [ ] **구체적인 제약사항**: 기술 스택, 코드 스타일, 요구사항

## 💡 참고: 좋은 프롬프트 예시

### 예시 1: 기능 구현
```
Python으로 이메일 검증 함수를 만들어줘.

요구사항:
- 함수명: validate_email
- 입력: 문자열 (이메일 주소)
- 출력: bool (유효하면 True, 아니면 False)
- 검증 규칙:
  * @ 기호가 정확히 하나
  * @ 앞뒤로 문자 존재
  * 도메인에 .이 최소 하나

예시:
- validate_email("test@example.com") → True
- validate_email("invalid") → False
- validate_email("@example.com") → False

외부 라이브러리 없이 기본 문법만 사용해줘.
```

### 예시 2: 버그 수정
```
다음 코드에서 ZeroDivisionError가 발생해:

```python
def calculate_average(numbers):
    return sum(numbers) / len(numbers)

result = calculate_average([])
```

에러 메시지:
```
ZeroDivisionError: division by zero
```

빈 리스트가 입력되면 0을 반환하도록 수정해줘.
테스트 코드도 함께 작성해줘.
```

### 예시 3: 리팩토링
```
다음 코드를 리팩토링해줘:

```python
def process_data(data):
    result = []
    for item in data:
        if item > 0:
            result.append(item * 2)
    return result
```

개선하고 싶은 점:
1. 더 pythonic하게 (list comprehension 사용)
2. 타입 힌트 추가
3. docstring 추가

기능은 동일하게 유지해줘.
```
