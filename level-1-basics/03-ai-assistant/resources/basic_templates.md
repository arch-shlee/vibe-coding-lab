# 기본 프롬프트 템플릿

## 1. 새 기능 구현

```
[프로젝트 설명]
나는 [프로젝트명]을 만들고 있어.

[목표]
[기능명] 기능을 구현하고 싶어.

[요구사항]
- 요구사항 1
- 요구사항 2
- 요구사항 3

[기술 스택]
- 언어/프레임워크:
- 라이브러리:

[제약사항]
- 제약 1
- 제약 2

[예시]
```(언어)
(사용 예시 코드)
```

[추가 요청]
- 주석 포함
- 테스트 코드 작성
- ...
```

### 실제 예시

```
Python으로 이메일 검증 함수를 만들어줘.

요구사항:
- 함수명: validate_email
- 입력: 문자열 (이메일 주소)
- 출력: bool (유효하면 True)
- 검증 규칙:
  * @ 기호가 정확히 하나
  * @ 앞뒤로 문자 존재
  * 도메인에 .이 최소 하나

제약사항:
- 외부 라이브러리 없이 기본 문법만
- 타입 힌트 포함
- docstring 작성

예시:
```python
validate_email("test@example.com")  # True
validate_email("invalid")  # False
```
```

```

[시그니처]
- 함수명/클래스명:

- 반환값:

```(언어)

```

[요청사항]
- 타입 힌트

### 실제 예시

```

- 함수명: unique_and_sort
- 입력: List[int]
- 반환: List[int]

동작:
1. 중복 제거
2. 오름차순 정렬
3. 정렬된 리스트 반환

예시:
```python
result = unique_and_sort([3, 1, 2, 1, 3])

요청사항:
- 타입 힌트 포함
- docstring 작성 (Google 스타일)
- 빈 리스트 처리

## 4. 테스트 작성

```
[테스트 대상]
다음 [함수/클래스]에 대한 테스트를 작성해줘:

```(언어)
(테스트할 코드)
```

[테스트 케이스]
1. 정상 케이스:
2. Edge case:
3. 에러 케이스:

[테스트 프레임워크]
- pytest / unittest / ...

[요청사항]
- 각 케이스마다 테스트 함수
- 명확한 테스트 이름
- Given-When-Then 주석
```

### 실제 예시

```
다음 divide 함수에 대한 pytest 테스트를 작성해줘:

```python
def divide(a: float, b: float) -> float:
    if b == 0:
        raise ValueError("0으로 나눌 수 없습니다")
    return a / b
```

테스트 케이스:
1. 정상 케이스: 10 / 2 = 5
2. 소수점: 10 / 3 = 3.333...
3. 음수: -10 / 2 = -5
4. 0으로 나누기: ValueError 발생

요청사항:
- 각 케이스마다 별도 테스트 함수
- 테스트 함수명은 test_divide_로 시작
- pytest.raises 사용하여 예외 테스트
```

## 5. 리팩토링

```
[리팩토링 대상]
다음 코드를 리팩토링해줘:

```(언어)
(현재 코드)
```

[개선 목표]
- 가독성 향상
- 성능 개선
- 유지보수성
- ...

[구체적 요청]
- (예: list comprehension 사용)
- (예: 중복 제거)
- (예: 함수 분리)

[제약사항]
- 기능은 동일하게 유지
- 호환성 유지
- ...

[추가 요청]
리팩토링 전후를 비교하고 개선사항을 설명해줘.
```

### 실제 예시

```
다음 코드를 더 pythonic하게 리팩토링해줘:

```python
def get_even_numbers(numbers):
    result = []
    for num in numbers:
        if num % 2 == 0:
            result.append(num)
    return result
```

개선 목표:
- 더 간결하게 (list comprehension)
- 타입 힌트 추가
- docstring 추가

제약사항:
- 기능은 동일하게 유지
- 기존 호출 코드는 수정 없이 작동

추가 요청:
개선된 점을 설명해줘.
```
