# 학습 프롬프트 템플릿

## 1. 개념 설명

```
[개념]
[개념명]에 대해 설명해줘.

[대상]
나는 [수준] 수준이야.
(예: 초보자, 중급자, 다른 언어 경험자)

[원하는 설명 방식]
- 쉬운 비유로
- 실제 사용 예시와 함께
- 다른 개념과 비교해서
- 단계별로

[특히 알고 싶은 것]
- (구체적인 질문)
```

### 실제 예시

```
Python의 데코레이터(decorator)에 대해 설명해줘.

대상:
나는 Python 기본 문법은 아는 초보자야.
함수, 클래스는 이해하고 있어.

원하는 설명 방식:
- 데코레이터가 왜 필요한지부터
- 간단한 예시로 시작해서
- 점점 복잡한 예시로
- 실제로 어디에 쓰이는지

특히 알고 싶은 것:
- @기호는 뭐야?
- 함수를 인자로 받는다는 게 무슨 뜻?
- 언제 사용하면 좋을까?
```

## 2. 코드 설명

```
[코드]
다음 코드를 설명해줘:

```(언어)
(설명받을 코드)
```

[설명 방식]
- 한 줄씩 단계별로
- 전체 흐름을 먼저
- 핵심 개념만 간단히

[특히 설명해줘]
- (이해 안 되는 부분)
- (낯선 문법)

[배경]
나는 [언어/개념]은 알지만,
[특정 부분]은 처음이야.
```

### 실제 예시

```
다음 코드를 설명해줘:

```python
def fibonacci(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fibonacci(n-1, memo) + fibonacci(n-2, memo)
    return memo[n]
```

설명 방식:
전체 흐름을 먼저 설명하고,
각 부분이 왜 필요한지 설명해줘.

특히 설명해줘:
- memo={}가 기본 인자인데 왜 문제가 안 될까?
- 이게 메모이제이션이라는 건 알겠는데,
  어떻게 작동하는지 단계별로

배경:
재귀는 알지만 메모이제이션은 처음이야.
```

## 3. 개념 비교

```
[개념들]
[개념 A]와 [개념 B]의 차이를 설명해줘.

[알고 싶은 것]
- 각각 무엇인지
- 어떤 차이가 있는지
- 언제 무엇을 쓰는지
- 장단점 비교

[예시]
실제 사용 예시도 함께 보여줘.
```

### 실제 예시

```
Python의 리스트와 튜플의 차이를 설명해줘.

알고 싶은 것:
- 문법적 차이
- 성능 차이
- 사용 목적 차이
- 메모리 차이

예시:
각각 어떤 상황에서 쓰면 좋은지
실제 코드로 보여줘.

배경:
둘 다 여러 값을 저장한다는 건 알겠는데,
언제 뭘 써야 할지 모르겠어.
```

## 4. 문제 해결 방법

```
[문제]
(해결하고 싶은 문제)

[제약사항]
- (사용 가능한 도구/라이브러리)
- (불가능한 것)

[원하는 답변]
- 여러 접근 방법
- 각 방법의 장단점
- 추천하는 방법과 이유

[배경]
나는 (수준), (알고 있는 것)
```

### 실제 예시

```
문제:
대용량 로그 파일(10GB)에서
특정 에러 메시지를 찾고 싶어.

제약사항:
- Python 사용
- 메모리는 제한적 (2GB)
- 빠를수록 좋음

원하는 답변:
- 여러 접근 방법 제시
- 각 방법의 성능/메모리 비교
- 추천 방법과 코드 예시

배경:
Python 기본은 알지만,
대용량 파일 처리 경험은 없어.
```

```

[주제]의 모범 사례(best practices)를 알려줘.

[구체적으로]
- 꼭 지켜야 할 것
- 피해야 할 것
- 일반적인 실수
- 팁과 트릭

[예시]
좋은 예시와 나쁜 예시를 비교해서 보여줘.
```

### 실제 예시

```
Python 함수 작성의 모범 사례를 알려줘.

구체적으로:
- 함수명 짓기
- 매개변수 설계
- 반환값 처리
- 에러 처리
- docstring 작성
- 타입 힌트

예시:
나쁜 예시와 좋은 예시를 비교해서
왜 좋은 예시가 더 나은지 설명해줘.
```

## 6. 단계별 학습

```
[목표]
[기술/개념]을 배우고 싶어.

[현재 수준]
나는 (알고 있는 것)

[학습 계획 요청]
다음을 포함한 학습 계획을 만들어줘:
- 학습 순서 (단계별)
- 각 단계에서 배울 내용
- 실습 과제
- 참고 자료

[목표 기간]
(원한다면) [기간] 안에 마스터하고 싶어.
```

### 실제 예시

```
목표:
FastAPI로 REST API를 만들 수 있게 되고 싶어.

현재 수준:
- Python 기본 문법: 알아
- 웹 개발: 처음
- API: 개념만 들어봤어

학습 계획 요청:
1단계부터 차근차근 학습 계획을 짜줘.
각 단계마다:
- 배울 내용
- 실습 과제
- 예상 소요 시간

특히:
- Hello World부터 시작해서
- 점점 복잡한 기능으로
- 마지막에는 실전 프로젝트까지
```

## 7. 에러 이해

```
[에러]
다음 에러 메시지가 무슨 뜻인지 설명해줘:

```
(에러 메시지)
```

[원하는 설명]
- 에러가 왜 발생하는지
- 일반적인 원인
- 해결 방법
- 예방 방법

[코드 컨텍스트]
(가능하면) 에러가 발생한 코드:
```(언어)
(코드)
```
```

### 실제 예시

```
다음 에러가 무슨 뜻인지 설명해줘:

```
RecursionError: maximum recursion depth exceeded
```

원하는 설명:
- 왜 이런 에러가 나는지
- "recursion depth"가 뭔지
- 어떻게 고치는지
- 재귀를 안전하게 쓰는 방법

코드 컨텍스트:
```python
def factorial(n):
    return n * factorial(n - 1)

result = factorial(5)
```

배경:
재귀 함수를 처음 써봤는데 에러가 나.
```

## 8. 실전 적용

```
[배운 것]
(개념/기술)을 배웠어.

[적용하고 싶은 상황]
이걸 실제 프로젝트에 어떻게 적용할 수 있을까?

[프로젝트]
- 종류: (웹앱, CLI 도구 등)
- 목적: (무엇을 하는지)
- 현재 상태: (어디까지 만들었는지)

[질문]
- 어디에 적용하면 좋을까?
- 구체적인 적용 예시
- 주의사항
```

### 실제 예시

```
배운 것:
Python 데코레이터를 배웠어.

적용하고 싶은 상황:
Flask 웹 애플리케이션에 적용하고 싶어.

프로젝트:
- 종류: Flask REST API
- 목적: TODO 앱 백엔드
- 현재: 기본 CRUD는 완성

질문:
- 데코레이터로 뭘 할 수 있을까?
  (로깅? 인증? 캐싱?)
- 각각의 구현 예시
- Flask에서 자주 쓰이는 데코레이터 패턴
```

## 📚 학습 체크리스트

새로운 개념을 배울 때:

- [ ] **정의**: 무엇인가?
- [ ] **이유**: 왜 필요한가?
- [ ] **사용법**: 어떻게 쓰는가?
- [ ] **예시**: 실제 예시는?
- [ ] **비교**: 비슷한 개념과 차이는?
- [ ] **주의사항**: 함정은?
- [ ] **실습**: 직접 해봤는가?

## 🔄 반복 학습 패턴

```
1. 간단히 설명 들기
   "클로저를 5줄로 설명해줘"

2. 예시로 이해하기
   "간단한 예시 하나만"

3. 깊이 파고들기
   "왜 이렇게 작동해?"

4. 비교하며 이해하기
   "다른 방법과 비교해줘"

5. 실습하기
   "연습 문제 내줘"

6. 실전 적용하기
   "내 프로젝트에 어떻게 쓸까?"
```
