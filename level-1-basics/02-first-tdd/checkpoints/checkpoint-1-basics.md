# Checkpoint 1: TDD 기본 이해

> TDD의 기본 개념을 이해했는지 확인하세요.

## 📚 개념 이해

### TDD란?

- [ ] TDD가 Test-Driven Development의 약자임을 안다
- [ ] "테스트를 먼저 작성하는 개발 방법"임을 이해한다
- [ ] 전통적 개발과의 차이점을 설명할 수 있다

### Red-Green-Refactor 사이클

- [ ] 🔴 Red 단계가 무엇인지 안다 (실패하는 테스트 작성)
- [ ] 🟢 Green 단계가 무엇인지 안다 (최소 구현)
- [ ] 🔵 Refactor 단계가 무엇인지 안다 (코드 개선)
- [ ] 왜 이 순서로 진행하는지 이해한다

### TDD의 장점

- [ ] 버그 조기 발견
- [ ] 안전한 리팩토링
- [ ] 명확한 요구사항
- [ ] 더 나은 설계

## 🧪 pytest 기본

### 설치 및 실행

- [ ] pytest가 설치되어 있다
- [ ] `pytest --version`으로 버전 확인 가능
- [ ] `pytest` 명령으로 테스트 실행 가능

### 기본 문법

- [ ] `test_` 로 시작하는 함수가 테스트임을 안다
- [ ] `assert` 문으로 검증함을 안다
- [ ] 테스트 실패 vs 통과를 구분할 수 있다
