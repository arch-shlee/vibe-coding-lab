# Claude 작업 가이드라인

> 이 프로젝트에서 Claude가 작업할 때 따라야 할 규칙들입니다.

## Git Commit 규칙

### Commit Message 작성

**일반 원칙**:
- Conventional Commits 스타일 사용
- 한글로 작성 (본문 설명)
- 명확하고 간결하게

**포맷**:
```
<type>: <subject>

<body>
```

**Type 종류**:
- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅
- `refactor`: 코드 리팩토링
- `test`: 테스트 코드
- `chore`: 빌드, 설정 파일 수정

### Co-Authored-By 규칙

**중요**: 이 프로젝트에서는 Co-Authored-By를 **사용하지 않습니다**.

```
❌ 사용하지 말 것:
Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

**이유**:
- 모든 commit은 사용자(arch-shlee)의 작업으로 기록
- Claude는 도구/조수 역할
- Git 히스토리의 일관성 유지

## 파일 작성 규칙

### 문서 작성

- **언어**: 한글 우선 (코드 주석도 한글)
- **톤**: 친근하고 이해하기 쉽게
- **구조**: 명확한 섹션 구분
- **예제**: 풍부한 코드 예제 포함

### 이미지 처리

- 이미지는 사용자가 직접 캡처
- 플레이스홀더만 포함:
  ```markdown
  <!-- 📸 TODO: 스크린샷 추가 - filename.png -->
  <!-- 설명: 화면 설명 -->
  ```

### 코드 스타일

**Python**:
- PEP 8 준수
- 타입 힌트 사용
- Docstring 필수 (Google 스타일)

## 작업 방식

### 협업 스타일

- **사용자 주도**: 방향은 사용자가 결정
- **Claude 지원**: 구현과 제안
- **피드백 기반**: 지속적인 개선

### TodoWrite 사용

- 복잡한 작업 시작 시 TodoWrite로 작업 추적
- 완료 시 즉시 상태 업데이트
- 한 번에 하나의 todo만 in_progress

## 프로젝트 특성

### 학습 플랫폼

- **초심자 중심**: 쉬운 설명
- **실습 중심**: hands-on 예제
- **단계적**: 점진적 난이도 증가

### 품질 기준

- **명확성**: 애매한 표현 지양
- **완성도**: 반쪽짜리 작업 지양
- **일관성**: 모든 모듈 동일한 구조
